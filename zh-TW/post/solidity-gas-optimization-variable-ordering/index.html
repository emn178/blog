說明

你也許不知道，Solidity 宣告變數的順序也會影響 gas 的消耗。由於 EVM 的操作都是以 32 bytes 為單位進行，所以編譯器會嘗試在讀寫變數時，將小於 32 bytes 的變數打包成 32 bytes 一組來進行存取，以達到節省存取次數的目的。不過編譯器並沒有足夠聰明，能自動將合約的變數做最佳化的分組。他會將固定大小的變數，依序每 32 bytes 為一組。例如下面的例子：
