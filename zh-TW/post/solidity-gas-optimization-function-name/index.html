結論

在暸解了這些運作原理之後，我們可以做出以下兩種應對方式：

  1. 減少公用成員
  2. 將常用函式順位提高

第一點很容易理解與執行，而第二點我們可以透過命名的方式來達成。但是要人工的方式，命名到雜湊值剛好較低的方式並不容易。於是我寫了一個函式名稱優化小工具來處理這個問題。以上面的例子來說，假設 b() 是常用的函式，若想將它提升順位，可以輸入函式簽章 b()，程式會自動找出一個 Method ID 前兩個 bytes 為零，結尾亂碼的新名稱：
